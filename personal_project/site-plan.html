Site Plan

Purpose & Audience

Purpose

The purpose of this site is to help people plan out their workouts before they go to the gym. A lot of people show up to the gym without really knowing what they're going to do, so this site lets you pick exercises, put them in a list, and then print it out so you have something to follow. It's meant to be simple and easy to use.

Audience

This site is for pretty much anyone who works out or wants to start working out. It doesn't matter if you're a beginner or have been going to the gym for years. I wanted it to work for all fitness levels so I included strength, cardio, bodyweight, and stretching exercises. Basically if you exercise or want to, this site is for you.

Dynamic Elements (JavaScript)

Here is how I plan to use JavaScript to make the site interactive:

  • Arrays & Objects: I'm going to store all the exercises in an array of objects. Each object will have things like the exercise name, type, muscleGroup, sets, reps, and a short desc. There will also be a second array called currentPlan that keeps track of whatever exercises the user has added to their plan.
  • Array methods: I'll use .filter() to only show exercises that match whatever button the user clicked. I'll use .forEach() to loop through the exercises and build the cards on the page. I also plan to use .map() somewhere in my template functions.
  • DOM interaction: The filter buttons will have event listeners on them so when you click one it updates what exercises show up. Clicking "Add to Plan" on a card will add that exercise to the plan list on the side of the page.
  • Conditional branching: I'll use if/else statements to check things like whether the plan is empty (so I can show a message like "No exercises added yet"), and whether an exercise is already in the plan so it doesn't get added twice.
  • Multiple functions: The plan is to have several functions including init(), filterExercises(), renderExercises(), cardTemplate(), addToPlan(), renderPlan(), removeFromPlan(), and printPlan().
  • Print feature: Instead of using a complicated library, the print button will just call window.print() which is built into every browser. I'll use a @media print rule in CSS to hide everything except the plan so it prints cleanly.

Logo

For the logo I'm going with a simple blue rounded square with an image that resembles a barbell, next to the site name. I might make a nicer version using Canva but this gets the idea across. The font I picked for the name is Barlow Condensed which I think looks pretty athletic.

Content

Page Sections

  • Header: The logo, the site name "FitPlan", and the tagline "Plan it. Print it. Crush it."
  • Hero: A short description of what the site does and a picture of someone at the gym.
  • Filter buttons: Buttons for workout type (Strength, Cardio, Bodyweight, Stretch) and muscle group (Chest, Back, Legs, Shoulders, Arms, Core, All).
  • Exercise cards: These get created by JavaScript. Each one shows the exercise name, what type it is, how many sets and reps, a quick description, and an "Add to Plan" button.
  • My Plan: A section that shows the exercises you've picked. If it's empty it'll say something like "No exercises added yet." There will also be a button to remove exercises and a button to print the plan.
  • Footer: Just a simple footer with the tagline and my name.
